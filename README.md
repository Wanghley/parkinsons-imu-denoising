## About The Project

This project implements and compares three denoising autoencoder architectures for 1D time-series signals. Given a clean signal **x**, a corrupted version **x_noisy** is generated by injecting artificial noise. A neural network **f_θ** is trained to reconstruct the original:

```
x̂ = f_θ(x_noisy)
```

The model minimizes the mean-squared reconstruction loss:

```
L(θ) = E[‖x − f_θ(x_noisy)‖²]
```

Beyond basic denoising, the project systematically studies how architectural choices (FC vs. CNN vs. LSTM), latent dimension size, and noise type affect denoising performance.

## Usage

Run all four experiments sequentially:
```sh
python main.py
```

Run a specific experiment:
```sh
python main.py --exp arch        # Architecture comparison (FC vs CNN vs LSTM)
python main.py --exp latent      # Latent dimension sweep
python main.py --exp noise       # Noise robustness across noise levels
python main.py --exp hyperparam  # Learning rate × batch size grid search
```

Results (plots and JSON metrics) are saved to the `results/` directory.

## Model Architectures

All models follow an encoder–decoder structure with a bottleneck latent vector **z**:

```
z = Encoder(x_noisy)     x̂ = Decoder(z)
```

| Architecture | Parameters (d_z=64) | Inductive Bias |
|---|---|---|
| **FC Autoencoder** | ~345K | None — treats signal as flat vector |
| **1D-CNN Autoencoder** | ~1.36M | Local temporal patterns via shared conv kernels |
| **LSTM Autoencoder** | ~818K | Long-range sequential dependencies |

### FC Autoencoder
Encoder: `512 → 256 → 128 → d_z` (Linear + ReLU)  
Decoder: mirrors encoder with a final linear layer outputting 512 values.

### 1D-CNN Autoencoder
Encoder: four strided Conv1d stages (×2 spatial compression each) → FC bottleneck.  
Decoder: four ConvTranspose1d stages → FC projection to signal length.

### LSTM Autoencoder
Encoder: 2-layer bidirectional LSTM → temporal mean-pool → linear projection to `d_z`.  
Decoder: broadcast latent vector over L steps → unidirectional LSTM → linear output.

See [`report/student2_report.md`](report/student2_report.md) for full architecture diagrams and parameter counts.

## Noise Models

Four noise corruption functions are implemented in [`noise.py`](noise.py):

| Noise Type | Description |
|---|---|
| **Gaussian** | Additive white noise: `x_noisy = x + ε`, `ε ~ N(0, σ²)` |
| **Random masking** | Zeros out randomly selected contiguous segments |
| **Impulse noise** | Inserts sparse random-amplitude spikes at random positions |
| **Sinusoidal interference** | Adds a periodic tone to simulate background interference |

## Experiments

### 1. Architecture Comparison
Trains FC, CNN, and LSTM under identical conditions (Gaussian σ=0.1, lr=1e-3, batch=64, 100 epochs) and reports MSE, output SNR, and SNR improvement on a held-out test set.

### 2. Latent Dimension Sweep
Sweeps `d_z ∈ {8, 16, 32, 64, 128, 256}` for each architecture, revealing the compression–quality tradeoff.

### 3. Noise Robustness
Trains on σ=0.1 Gaussian noise, then evaluates on σ ∈ {0.05, 0.1, 0.2, 0.4}. Measures generalization to out-of-distribution noise intensities.

### 4. Hyperparameter Grid Search
Searches learning rates `{1e-4, 5e-4, 1e-3}` × batch sizes `{32, 64, 128}` using the CNN architecture to find the optimal training configuration.

## Evaluation Metrics

**Mean Squared Error (MSE):**
```
MSE = (1 / N·L) · Σ (x_it − x̂_it)²
```

**Signal-to-Noise Ratio (SNR, dB):**
```
SNR(x, x̂) = 10 · log10( ‖x‖² / ‖x − x̂‖² )
```

**SNR Improvement (SNRi):**
```
SNRi = SNR(x, x̂) − SNR(x, x_noisy)
```
A positive SNRi confirms the model is actively denoising rather than simply passing the noisy input through.
